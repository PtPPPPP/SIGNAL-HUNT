# SIGNAL HUNT 使用文档

本文面向展会管理员和现场工作人员，说明如何完成活动配置、现场抽奖、兑奖和数据备份。

## 1. 打开系统

Windows 桌面版启动后会自动进入展会大屏。首次使用且尚未创建活动时，系统会同时打开后台配置窗口。

桌面版常用快捷键：

- `Ctrl + Shift + A`：打开或切换到后台控制窗口。
- `Ctrl + Shift + S`：打开或切换到工作人员窗口。
- `Ctrl + Shift + E`：结束当前中奖结果，令大屏返回待机。

### 选择大屏显示模式

在后台打开“系统设置”，可为展会大屏选择：

- 窗口模式：普通可移动、可缩放窗口。
- 全屏模式：大屏占满当前显示器。
- 展会锁定模式（Kiosk）：正式布展使用的锁定大屏。

点击“应用显示模式”后立即作用于展会大屏。后台和工作人员使用的控制窗口不会进入
全屏或 Kiosk。选择保存在 Electron 用户数据目录，下次启动继续使用；该设置不属于
抽奖数据库，也不会随业务备份导入或导出。

开发环境可运行：

```powershell
npm install
npm run dev
```

然后使用以下页面：

- 大屏：`http://127.0.0.1:5180/display`
- 后台：`http://127.0.0.1:5180/admin/dashboard`
- 工作人员端：`http://127.0.0.1:5180/staff`

## 2. 首次配置

### 创建奖品

1. 在后台打开“奖品管理”。
2. 点击“新增奖品”。
3. 填写奖项名称、简称、等级、总量、剩余数量和基础权重。
4. 勾选“启用”，选择中奖策略，然后点击“保存奖品”。
5. 重复以上步骤，直到所有奖品都出现在奖品列表中。

奖品只有在“已启用、剩余库存大于 0、有效权重大于 0”时才会进入抽奖池。系统没有“谢谢参与”，每次有效抽奖都会抽中一个仍在奖池中的奖品。

### 设置中奖概率

1. 打开“概率与发放策略”。
2. 在简单模式中填写各奖项的中奖百分比。
3. 确认合计为 `100%`，并检查页面显示的预计中奖人数和库存风险。
4. 点击“保存并应用”。

新设置只影响下一次尚未开始的抽奖，不会改变已经生成的中奖结果。

### 创建并激活活动

1. 打开“活动管理”。
2. 填写活动名称和活动代码；开始、结束时间可以留空。
3. 点击“创建活动（草稿）”。
4. 在活动列表中找到该活动，点击“激活”。

同一台设备只能有一个已激活活动。激活新活动时，如果已有活动正在进行，系统会要求先确认暂停旧活动。

## 3. 现场抽奖与兑奖

1. 参与者在大屏点击抽奖按钮。
2. 系统先保存中奖结果并扣减库存，再播放动画和展示奖项。
3. 工作人员核对大屏与工作人员端显示的奖项。
4. 奖品发放完成后，在工作人员端点击“确认兑奖”。
5. 点击“结束当前结果并返回待机”，让下一位参与者继续抽奖。

“确认兑奖”和“结束展示”是两个独立操作，执行其中一个不会自动执行另一个。刷新、崩溃或重新打开页面后，已经提交但尚未结束的结果会恢复显示，不会重新抽奖或重复扣减库存。

如果记录需要作废，先填写作废原因，再点击“作废记录”并确认。作废不会自动恢复奖品库存；已兑奖记录不能直接作废。

## 4. 暂停或结束活动

- 临时停止抽奖：在“活动管理”中点击“暂停”。已提交的中奖结果会保留。
- 活动正式结束：点击“结束”。大屏会进入活动结束状态，历史记录和库存数据会保留。

## 5. 备份与恢复

在活动开始前和结束后，都应打开“系统管理”并点击“下载完整备份”。备份包含活动、奖品、抽奖记录和抽奖会话。

恢复备份时：

1. 将备份 JSON 放入“备份 JSON”输入区。
2. 检查解析出的备份时间和数据数量。
3. 点击“恢复备份”，再点击“确认恢复”。

恢复会完全替换当前数据。系统会在恢复前自动保存一次快照，必要时可点击“回滚到恢复前”。

## 6. 遇到问题时

- 大屏显示“尚未配置活动”：检查是否已经创建并激活活动。
- 大屏显示“系统已暂停”：在“活动管理”中重新激活活动。
- 无法继续抽奖：检查是否存在已启用、仍有库存且有效权重大于 0 的奖品。
- 操作失败或状态异常：打开 `/diagnostics` 查看现场自检和诊断日志；不要清空浏览器数据，否则本机数据会丢失。
